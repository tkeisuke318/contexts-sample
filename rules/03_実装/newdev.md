# 実装フェーズ — 新規開発

## 遵守事項

- コーディング規約に則り実装すること
- 方式設計で定義された依存方向・責務分離・共通方式に則り実装すること

## 禁止事項

- 機密情報（パスワード・APIキー・接続文字列）をソースコードにハードコードしてはならない
- SQLインジェクションが発生しうる文字列連結でSQLを組み立ててはならない
- ユーザー入力値をバリデーションなしに処理してはならない
- 出力値をエスケープせずにHTMLに埋め込んではならない
- デバッグ用コメントアウトコードをコミットしてはならない

## レビュー観点マーカー

インプットに根拠がない記述は、値の直後にインラインでマーカーを付ける。マーカーなしの記述は「インプットに根拠がある」とみなす。

| マーカー | 意味 |
|---------|------|
| `※推測` | 文脈から推論した。根拠はあるが明示されていない |
| `※要確認` | 情報不足で判断できない。仮置きしている |

```
# Good
クラス名: OrderService ※推測

# Bad（末尾まとめは不可）
クラス名: OrderService
※ 上記は推測です
```
