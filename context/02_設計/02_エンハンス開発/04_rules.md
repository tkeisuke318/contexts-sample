# 設計フェーズ（エンハンス開発）— 規約

## 規約

- **差分マーカー**：変更箇所には `--- 追加 ---` / `--- 変更 ---` / `--- 削除 ---` を付与する
- **後方互換性タグ**：API・データ形式の互換破壊変更には `[互換破壊]` タグを付与し、移行方針を記述する
- **マイグレーション要否**：DB 変更を含む設計書にはマイグレーション要否フラグを付与する
- **設計書バージョン**：更新した設計書のヘッダにバージョン番号と変更日を記載する
