# 要件定義フェーズ（エンハンス開発）— 作業手順

## 作業手順

1. 変更要求書をインプットに、既存の要件一覧・ユースケースを参照して変更対象の要件を特定する
2. 変更対象要件に関連する既存要件（依存・参照関係）を洗い出し、影響範囲を確定する
3. 変更・追加・削除する要件を区分し、既存 ID はそのまま維持して差分マーカーを付与する
4. 影響を受ける既存ユースケースを特定し、フローの変更差分（追加・変更・削除ステップ）を記述する
5. 変更要件と既存非機能要件（性能・可用性・セキュリティ）の整合性を確認する
6. 変更による既存機能の後方互換性（API・データ形式・画面 I/F）への影響を評価する
7. 変更後の要件一覧・ユースケース（差分マーカー付き）を確定し、設計フェーズへ引き継ぐ
