# 設計フェーズ — リバース

## 規約

- **レイヤー依存方向**：Controller → Service → Repository のみ許可（逆方向・横断禁止）

## 禁止事項

- 要件一覧に存在しない機能を設計書に追加してはならない
- マスタデータになるべき値（区分・コードなど）を画面に直接ハードコードしてはならない
- 外部システム連携設計書なしに外部システム連携の仕様を画面設計書・テーブル定義書に記述してはならない
- Controllerにビジネスロジックを記述してはならない
- Repositoryにビジネスロジックを記述してはならない
- SQLをService層に直接記述してはならない

## レビュー観点マーカー

リバースは「実装が正」であるため、マーカーの意味が新規開発と異なる。マーカーなしの記述は「実装から明確に読み取れた事実」とみなす。

| マーカー | 意味 |
|---------|------|
| `※推測` | 実装から読み取れたが意図が不明確。別の解釈の余地がある |
| `※要確認` | 実装を読んでも判断できない。ユーザーへの確認が必須 |
| `※実装と乖離の可能性` | 古いコードや複数箇所に矛盾があり、事実確認が必要 |

```
# Good
更新日時: updated_at ※推測（カラム名から推測。updated_atが論理削除を兼ねている可能性あり）
承認フロー: 不明 ※要確認（コードに承認ロジックが見当たらない）

# Bad
更新日時: updated_at
（なぜそう判断したかわからない）
```

## 読み込み元

| 成果物の種別 | 主な読み込み元 |
|------------|-------------|
| 画面一覧・画面遷移図 | フロントエンドのルーティング定義・画面コンポーネント一覧 |
| 画面設計書 | 画面テンプレート・フロントエンドコード |
| ER図・テーブル一覧・テーブル定義書 | DDL・マイグレーションファイル・既存DB |
| API一覧・API設計書 | ルーティング定義・コントローラー実装 |
| クラス設計書・CRUD図 | ソースコード（Controller・Service・Repository） |
| バッチ一覧・バッチ設計書 | バッチ処理ソースコード・ジョブスケジューラ設定 |
| 帳票一覧・帳票設計書 | 帳票出力コード・帳票テンプレートファイル |
| コード定義一覧・コード定義詳細 | Enum定義・定数ファイル・マスタテーブル |
| 外部システム連携一覧・外部システム連携設計書 | 外部連携クライアントコード・設定ファイル |
| システム構成 | インフラ構成ファイル（IaC・サーバー設定・ネットワーク設定） |
