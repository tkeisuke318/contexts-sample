# 設計フェーズ（エンハンス開発）— 作業手順

## 作業手順

1. 変更要件一覧をインプットに、変更対象の既存設計書（画面・API・DB・クラス等）を特定する
2. 変更対象設計書の現状を確認し、変更・追加・削除する設計要素と影響範囲を洗い出す
3. 変更箇所を差分マーカー（`--- 追加 ---` / `--- 変更 ---` / `--- 削除 ---`）付きで設計書に反映する
4. 変更によって影響を受ける既存設計要素（API・DB・クラス・画面）を特定し、整合性を確認する
5. API・データ形式の後方互換性（バージョニング・非破壊変更）の設計方針を決定する
6. 変更後のシーケンス図・ER 図等を差分ベースで更新する
7. 変更設計書を設計レビュー後、実装フェーズへ引き継ぐ
