# 設計フェーズ — 新規開発

## 規約

- **レイヤー依存方向**：Controller → Service → Repository のみ許可（逆方向・横断禁止）

## 禁止事項

- 要件一覧に存在しない機能を設計書に追加してはならない
- マスタデータになるべき値（区分・コードなど）を画面に直接ハードコードしてはならない
- 外部システム連携設計書なしに外部システム連携の仕様を画面設計書・テーブル定義書に記述してはならない
- Controllerにビジネスロジックを記述してはならない
- Repositoryにビジネスロジックを記述してはならない
- SQLをService層に直接記述してはならない

## レビュー観点マーカー

インプットに根拠がない記述は、値の直後にインラインでマーカーを付ける。マーカーなしの記述は「インプットに根拠がある」とみなす。

| マーカー | 意味 |
|---------|------|
| `※推測` | 文脈から推論した。根拠はあるが明示されていない |
| `※要確認` | 情報不足で判断できない。仮置きしている |

```
# Good
テーブル名: orders ※推測

# Bad（末尾まとめは不可）
テーブル名: orders
※ 上記は推測です
```
