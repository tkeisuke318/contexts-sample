# エンハンス 作業手順

## Step 1: フェーズと変更内容の確定

引数からフェーズ名（例: `01_要件定義`）を取得する。指定がない場合はユーザーに確認する。
以下を読み込む。

```
rules/{PHASE}/artifacts.md
rules/{PHASE}/enhance.md
```

`rules/{PHASE}/enhance.md` の「作業手順補足」をもとに、ユーザーの指示から以下を確定する。指示が曖昧な場合は確認してから進む。

- 変更対象の成果物（ファイルパス）
- 変更内容（追加・修正・削除のいずれか）
- 変更理由・背景

## Step 2: 既存成果物の読み込み

`rules/{PHASE}/enhance.md` の「作業手順補足 — 読み込み元」に従い、変更対象の成果物を読み込む。

対象ファイルが存在しない場合は作業を中断してユーザーに報告する。

## Step 3: 影響範囲の確認

`rules/{PHASE}/enhance.md` の「作業手順補足 — 影響範囲の観点」に従い、必要に応じて `rules/{PHASE}/artifacts.md` も参照して波及先を確認する。

影響がある場合は変更前にユーザーに報告し、対応方針を確認する。

## Step 4: 変更の実施

既存成果物に変更を加える。`rules/{PHASE}/enhance.md` の規約・禁止事項を遵守し、レビュー観点マーカーをインラインで付与する。マーカーなしの記述は「変更前から変わっていない」とみなされる。

フェーズ固有の注意事項（新規成果物のテンプレート利用など）は `rules/{PHASE}/enhance.md` の「作業手順補足」を参照する。

## Step 5: 完了報告

変更後、以下をユーザーに報告する。

1. 変更したファイルパス
2. 変更箇所の一覧（変更前→変更後）
3. `※推測` と `※要確認` の箇所一覧
4. 影響が見込まれる成果物の一覧（対応要否を含む）
