## HTTPステータスコード

> ★★ 使用するステータスコードを定義する。例外処理方式と整合させること

| ステータス | 意味 | 用途 |
|:---------:|------|------|
| 200 OK | 成功（参照・更新） | GET / PUT / PATCH 成功 |
| 201 Created | 作成成功 | POST 成功 |
| 204 No Content | 成功（本文なし） | DELETE 成功 |
| 400 Bad Request | 入力エラー | バリデーション違反 |
| 401 Unauthorized | 未認証 | トークン不正・欠落 |
| 403 Forbidden | 認可エラー | 権限不足 |
| 404 Not Found | リソース未発見 | |
| 409 Conflict | 競合 | 重複・楽観ロック失敗 |
| 429 Too Many Requests | レート制限超過 | |
| 500 Internal Server Error | システムエラー | 想定外例外 |
| 503 Service Unavailable | サービス停止中 | メンテナンス・過負荷 |

詳細は `04_例外処理方式.md` を参照。

## 共通レスポンス形式

> ★★ 成功時のレスポンス形式を定義する

```json
{
  "data": {
    "★★": "★★業務データ"
  },
  "meta": {
    "timestamp": "2026-04-11T12:00:00Z",
    "traceId": "★★リクエスト追跡ID"
  }
}
```

一覧取得時：

```json
{
  "data": [
    { "★★": "★★" }
  ],
  "meta": {
    "page": 1,
    "perPage": 25,
    "totalCount": 100,
    "totalPages": 4,
    "timestamp": "2026-04-11T12:00:00Z",
    "traceId": "★★"
  }
}
```
