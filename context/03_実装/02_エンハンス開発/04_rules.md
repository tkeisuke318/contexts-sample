# 実装フェーズ（エンハンス開発）— 規約

## 規約

- **最小変更の原則**：変更設計書に含まれる変更のみを実装する。設計外の「改善」は別チケットにする
- **ブランチ名**：`feature/[チケットID]-[機能名]` または `fix/[チケットID]-[内容]`
- **コミットメッセージ**：`[チケットID] 変更内容の概要`（変更種別を先頭に付ける：add/change/remove/fix）
- **既存テストのパス確認**：マージ前に既存テストをすべてパスさせること
