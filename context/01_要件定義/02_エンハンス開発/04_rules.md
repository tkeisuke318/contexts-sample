# 要件定義フェーズ（エンハンス開発）— 規約

## 規約

- **要件 ID の維持**：既存要件の ID は変更・削除時もそのまま維持する。新規追加は既存連番の続きを採番する
- **差分マーカー**：変更箇所には `--- 追加 ---` / `--- 変更 ---` / `--- 削除 ---` を付与する
- **後方互換性の明示**：API・データ形式・画面 I/F への互換破壊が発生する場合は要件に `[互換破壊]` タグを付与する
- **変更理由の注記**：差分マーカーの付いた要件には変更理由を 1 行で注記する
