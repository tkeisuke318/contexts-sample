# contexts-sample

システム開発の各フェーズにおいて、AIエージェントが成果物を生成するための**コンテキスト定義**と**ドキュメントテンプレート**のサンプルリポジトリです。

## コンセプト

**コンテキスト**とは、AIエージェントが成果物を作るための作業指示です。インプット・アウトプット・手順・制約を定義し、GraphRAGによって関連するノードが動的にシステムプロンプトへ注入される仕組みを前提に設計されています。

詳細は [concepts.md](./concepts.md) を参照してください。

## リポジトリ構成

```
contexts-sample/
├── context/          # 各フェーズのコンテキスト定義（AIエージェント向け作業指示）
├── templates/        # 各フェーズの成果物テンプレート
├── docs/             # 成果物の格納先（初期状態は空）
├── old_documents/    # 廃版・旧バージョン格納先
├── .claude/skills/   # Claude Code スキル定義（/pr など）
└── concepts.md       # コンテキスト設計のコンセプト説明
```

## コンテキストファイルの構成

各フェーズのコンテキストは `context/{フェーズ名}/` 配下の6ファイルに分割されています。

| ファイル | kind | 内容 |
|---------|------|------|
| `inout.md` | `inout` | インプット・アウトプット・使用インプットの定義 |
| `workflow.md` | `workflow` | 成果物を作るステップ |
| `knowhow.md` | `knowhow` | 失敗パターン・設計判断の背景など暗黙知 |
| `rules.md` | `rules` | 命名規則・採番ルール・構造規約 |
| `prohibit.md` | `prohibit` | 禁止パターン |
| `checklist.md` | `checklist` | フェーズ完了定義（DoD） |

各ファイルはYAMLフロントマター（`phase` / `kind`）を持ち、GraphRAGの型付きノードとして機能します。

## テンプレートの記法

- **`★★` 記法** — AIが埋めるべき入力項目の先頭に付与
- **図はMermaid形式** — フロー図・ER図・クラス図・シーケンス図はすべてMermaidで記述
