# 要求定義フェーズ（エンハンス開発）— 作業手順

## 作業手順

1. 変更要求（変更依頼書・チケット・RFP）から変更の目的とビジネス上の背景を確認する
2. 既存の要求定義書・ビジネスプロセス図を参照し、変更対象のスコープと影響範囲を特定する
3. 既存の業務課題一覧と照合し、変更による課題の解消・新規追加・変更を区別して整理する
4. 変更によって影響を受ける既存ビジネスプロセスを特定し、AS-IS との差分を明確にする
5. 変更要求と既存要求の整合性（矛盾・競合がないか）を確認する
6. 変更後のビジネス目標を定量化し、変更前との比較で効果を明示する
7. 差分を反映した要求定義書・課題一覧を更新し、変更箇所に `--- 変更 ---` / `--- 追加 ---` / `--- 削除 ---` マーカーを付与する
8. 更新した成果物をインプットに要件定義フェーズへ引き継ぐ
