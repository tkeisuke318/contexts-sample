# 移行設計フェーズ（エンハンス開発）— 作業手順

## 作業手順

1. エンハンス変更により影響を受けるデータ・スキーマの変更内容を確認する
2. **本番影響度（ダウンタイム・ロック時間・影響テーブル件数）を設計の冒頭に明示する**
3. 既存データに対するマイグレーション要否を判断し、必要な場合はデータ変換設計を行う
4. オンラインスキーマ変更（ゼロダウンタイム）の可否を判断し、設計に明記する
5. バックワード互換性が要件にある場合は段階的デプロイ（Blue-Green・フィーチャーフラグ）の設計を行う
6. ロールバック手順（変更前の状態に戻す方法）を設計する
7. 移行後の整合性確認クエリを設計する
