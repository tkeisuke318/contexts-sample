# 要求定義フェーズ（エンハンス開発）— 規約

## 規約

- **差分マーカー**：変更箇所には `--- 追加 ---` / `--- 変更 ---` / `--- 削除 ---` マーカーを必ず付与する
- **変更理由の記述**：差分マーカーの直後に変更理由を 1 行で注記する（例：`<!-- 変更理由：法改正対応 -->`）
- **既存要求の保持**：削除する要求はマーカーを付けたうえで残し、削除理由を記述する（バージョン管理のため即削除しない）
- **スコープ外の明記**：変更対象外の要求・業務プロセスを「今回スコープ外」セクションに列挙する
