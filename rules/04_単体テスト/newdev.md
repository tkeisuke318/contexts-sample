# 単体テストフェーズ — 新規開発

## 遵守事項

- テストの独立性：各テストケースは他のテストケースの実行結果に依存してはならない
- 外部依存：DB・外部APIはモックに置き換える
- カバレッジ基準：ブランチカバレッジ80%以上

## 禁止事項

- なし

## レビュー観点マーカー

インプットに根拠がない記述は、値の直後にインラインでマーカーを付ける。マーカーなしの記述は「インプットに根拠がある」とみなす。

| マーカー | 意味 |
|---------|------|
| `※推測` | 文脈から推論した。根拠はあるが明示されていない |
| `※要確認` | 情報不足で判断できない。仮置きしている |

```
# Good
テストケース名: 正常系_受注登録 ※推測

# Bad（末尾まとめは不可）
テストケース名: 正常系_受注登録
※ 上記は推測です
```
