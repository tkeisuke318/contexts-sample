## ログレベル

> ★★ 使用するログレベルと用途を定義する

| レベル | 用途 | 本番環境での出力 |
|--------|------|:----------------:|
| ERROR | ★★システム例外・業務継続不可 | ○ |
| WARN | ★★想定内だが注意が必要な状況 | ○ |
| INFO | ★★主要な処理の開始/終了・業務イベント | ○ |
| DEBUG | ★★開発時の詳細情報 | × |
| TRACE | ★★最詳細 | × |

---

## ログフォーマット

> ★★ ログの出力形式（JSON構造推奨）を定義する

```json
{
  "timestamp": "2026-04-11T12:00:00.000Z",
  "level": "INFO",
  "logger": "★★例：com.example.OrderService",
  "traceId": "★★リクエスト追跡ID",
  "userId": "★★操作ユーザーID",
  "message": "★★メッセージ",
  "context": {
    "★★keyName": "★★value"
  }
}
```

| フィールド | 必須 | 説明 |
|----------|:----:|------|
| timestamp | ○ | ISO8601形式 |
| level | ○ | ログレベル |
| logger | ○ | 出力元クラス |
| traceId | ○ | リクエスト単位の追跡ID |
| userId | △ | 認証済みの場合のみ |
| message | ○ | 本文 |
| context | △ | 任意の構造化情報 |

---

## 出力禁止項目

> ★★ ログに出してはいけない機密情報を列挙する

- ★★パスワード・APIキー・トークン
- ★★個人情報（氏名・住所・電話番号・クレジットカード番号等）
- ★★セッションID
