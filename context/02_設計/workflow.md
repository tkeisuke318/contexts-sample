# 設計フェーズ — 作業手順

## 作業手順

1. ユースケース一覧をもとに設計対象を確認し、各種一覧（画面・API・バッチ・テーブル・帳票）の骨格を作成する
2. 画面間の遷移をMermaidで図示する
3. 画面ごとに表示項目・入力項目・バリデーションを定義する
4. 業務データを整理し、エンティティと関係をMermaidで図示する（概念→論理→物理の順に精緻化）
5. ER図をもとにテーブル・カラム・型・制約を記述する
6. 外部システム連携がある場合は外部システム連携設計書・システム構成図を作成する
7. バッチがある場合はバッチ設計書を作成する
8. レイヤー構成をもとにクラスを洗い出し、依存・継承・実装関係をMermaidで図示する
9. 各クラスのpublicメソッドの引数・戻り値・処理概要を定義する
10. 主要ユースケースの処理フローをシーケンス図で図示する
11. エンドポイントごとにリクエスト・レスポンス・エラーを定義する
