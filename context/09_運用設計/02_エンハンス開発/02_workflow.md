# 運用設計フェーズ（エンハンス開発）— 作業手順

## 作業手順

1. エンハンス変更による運用手順への影響を特定する（監視設定・ジョブスケジュール・アラート等）
2. 既存の運用手順書・監視設定と変更内容を照合し、変更が必要な運用項目を洗い出す
3. 変更が必要な運用手順・監視設定を差分マーカー付きで更新する
4. 新規追加が必要な監視項目・アラート設定を設計する
5. エンハンス変更後の障害対応手順の変更有無を確認し、必要に応じて更新する
6. 更新した運用設計書を運用担当者と確認し、承認を得る
