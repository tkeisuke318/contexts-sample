# システムテストフェーズ — 作業手順

## 作業手順

1. システムテスト観点表を参照し、機能・性能・セキュリティ・可用性の観点でテスト計画を立案する
2. システムテスト計画書で要件トレーサビリティマトリクス・合格基準・スケジュールを確定する
3. 要件一覧の全要件IDに対して1件以上のテストケースを設計し、要件IDと紐付ける
4. ユースケース記述の正常フロー・代替フロー・例外フローを網羅するテストケースを作成する
5. 非機能要件定義書の目標値を基準値として、性能・セキュリティ・可用性テストの条件と手順を定義する
