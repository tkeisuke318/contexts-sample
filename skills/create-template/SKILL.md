docs/$0/templates/$1.md に新しいテンプレートドキュメントを作成してください。

作成前に `docs/` 配下の既存テンプレートを参照し、同じフェーズまたは類似フェーズのファイルと整合性を保ってください。

## 作成ルール

### ファイル先頭の2行（必須）

ファイルの最初の2行は以下の固定形式にする。`[ファイル名]` には実際のファイル名（拡張子込み）を入れる。

```
- このドキュメントは[ファイル名]のテンプレートです。
- ★★または> ★★ で始まる文章とその周辺は、このドキュメントを作成する際の指示文のため、指示として受け止め、最終成果物には残さないでください。
```

### 見出し構造

```
[先頭2行の箇条書き]

# ドキュメント名

---

## ドキュメント情報
## [本文セクション群]
## 変更履歴
```

### ドキュメント情報セクション（必須）

```markdown
## ドキュメント情報

> ★★ このドキュメントの管理情報（ID・日付・作成者・承認者）を記入する

| 項目 | 内容 |
|------|------|
| ドキュメントID | [PREFIX]-[連番4桁] |
| 対象★★[対象種別名] | ★★[対象名の説明] |
| 作成日 | ★★YYYY-MM-DD |
| 作成者 | ★★氏名 |
| 最終更新日 | ★★YYYY-MM-DD |
| 版数 | 1.0 |
| 承認者 | ★★承認者氏名 |
```

ドキュメントIDのプレフィックスはドキュメント種別を表す英字略称を使う（例：画面設計書→SCR、テーブル定義→DB-DEF）。

### セクション先頭の説明コメント（必須）

`##` および `###` の直後に、そのセクションで何を書くかを `> ★★ ` 形式で1行入れる。

```markdown
## セクション名

> ★★ このセクションに何を記述すればよいかを1文で説明する

[コンテンツ]
```

### ★★ マーカーのルール

| 付けるもの | 付けないもの |
|-----------|------------|
| プロジェクト固有の値（名称・数値・日付・説明文） | 列ヘッダー名・見出し・固定の凡例 |
| 例示（`★★説明（例：具体例）` の形式） | 連番フォーマット（`[連番4桁]`） |
| Mermaid図内のノード名・エッジラベル（識別子は除く） | Mermaid識別子（クラス名・テーブル名など） |

Mermaid識別子に ★★ を付けられない場合は、図の前に以下を入れる。

```
> ★★ この図全体を実際の[図の種類]に置き換える
```

### 図はすべてMermaid形式

フロー図・ER図・クラス図・シーケンス図・ガントチャートは必ずMermaid形式で記述する。

### 変更履歴セクション（必須）

```markdown
## 変更履歴

> ★★ ドキュメントの改版履歴を記録する。初版作成時は版数1.0、変更内容に「初版作成」と記入する

| 版数 | 変更日 | 変更者 | 変更内容 |
|------|--------|--------|---------|
| 1.0 | ★★YYYY-MM-DD | ★★氏名 | 初版作成 |
```

## 作成後の追加作業

テンプレートファイルを作成したら、対象フェーズの `context.md` のアウトプット表に新テンプレートへの参照を追記する。

```markdown
| [ドキュメント種別名] | `templates/$1.md` |
```
