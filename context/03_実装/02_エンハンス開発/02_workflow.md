# 実装フェーズ（エンハンス開発）— 作業手順

## 作業手順

1. 変更設計書（差分マーカー付き）を確認し、変更・追加・削除する実装箇所を特定する
2. 変更対象の既存コードを読み、変更前の動作・依存関係を把握してから実装に入る
3. 変更箇所を最小限に留め、スコープ外の既存コードを改変しない
4. API の後方互換性が設計で要求されている場合は、バージョニング・フィールド追加の実装を行う
5. 変更箇所に対するユニットテストを作成し、既存テストがすべてパスすることを確認する（リグレッション確認）
6. コードレビューを経て、変更内容をマージする
